### Detalhamento dos Laboratórios e Atividades

#### 1.1.1: Baixar recursos de aula

- Acessar a Plataforma de Treinamento: Navegue até o endereço fornecido para os materiais do curso (ex: LMS ou ambiente de treinamento).

- Localizar a Seção do Módulo 1: Encontre a pasta ou seção dedicada ao "Módulo 1: ServiceNow".

- Identificar os Arquivos: Localize a lista de arquivos de recursos (Guias de laboratório, datasets, etc.) e o conteúdo de prática.

- Realizar o Download: Clique no link ou ícone de download para salvar todos os recursos necessários em uma pasta local do seu computador.

- Verificar a Integridade: Abra os arquivos baixados para confirmar que estão acessíveis e não corrompidos.

#### 1.2.1: Conceder e verificar acesso a aplicativos

- Acessar o Painel de Administração: Faça login no seu ambiente de desenvolvimento/treinamento do ServiceNow como administrador.

- Navegar para Gerenciamento de Usuários: Use o Navegador de Aplicativos para ir para Usuários e Grupos -> Usuários.

- Localizar ou Criar um Usuário de Teste: Encontre o perfil de usuário que receberá o acesso ou crie um novo.

- Atribuir Perfis (Roles): Adicione os perfis necessários (ex: itil para ITSM, workspace_agent para Service Operations Workspace) ao usuário de teste.

- Verificar o Acesso: Faça logoff como administrador e faça login novamente com o usuário de teste. Verifique se ele tem acesso aos módulos e interfaces correspondentes (ex: se o link para o Workspace aparece).

#### 1.3.1: Criar incidentes usando listas e formulários clássicos
- Navegar para a Interface Clássica de ITSM: No Navegador de Aplicativos, vá para ITSM -> Incidentes -> Criar Novo.

- Preencher o Formulário:

- Preencha o campo Usuário Solicitante (Caller).

- Selecione uma Categoria e Subcategoria para o incidente.

- Defina a Prioridade com base na Urgência e Impacto.

- Insira um Breve Resumo (Short Description) e a descrição completa do problema.

- Salvar o Incidente: Clique em Enviar (Submit) para criar o registro.

- Verificar a Criação: Localize o número do incidente recém-criado na lista de incidentes abertos.

#### 1.3.2: Trabalhar em incidentes usando Service Operations Workspace
- Acessar o Service Operations Workspace: Navegue até a URL do Workspace ou use o atalho no Navegador de Aplicativos.

- Aceitar o Incidente: Na Caixa de Entrada (Inbox) ou na lista de Meus Trabalhos (My Work), clique no incidente criado na atividade anterior.

- Analisar o Problema: Utilize as guias de Contexto e Linha do Tempo no Workspace para revisar as informações do solicitante e o histórico.

- Atualizar o Status: Mude o Estado (State) do incidente para Em Andamento (In Progress).

- Adicionar uma Nota de Trabalho: Registre as ações tomadas ou as descobertas na seção Notas (Notes).

#### 1.3.3: Verifique o status do incidente usando o Portal do Centro de Funcionários
- Acessar o Employee Center Portal: Navegue até a URL do portal de autoatendimento.

- Fazer Login como Solicitante: Faça login como o usuário que você definiu como Usuário Solicitante (Caller) no passo 1.3.1.

- Navegar para Minhas Solicitações: Procure a seção Minhas Solicitações, Meus Itens ou Minhas Tarefas.

- Localizar o Incidente: Encontre o incidente que você trabalhou no passo anterior usando o número ou o breve resumo.

- Verificar o Status: Clique no incidente e confirme que o Estado (State) está refletindo a última atualização (ex: Em Andamento).
